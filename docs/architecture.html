01

6-Layer System Overview

Every request flows top-to-bottom through six layers. Memory write-back is async and never blocks the response path. Color key: ■ purple = input, gray = gateway, ■ teal = orchestrator, ■ amber = memory, ■ coral = tools, ■ blue = output.

02

Voice Request Flow

End-to-end trace from wake word detection to first audio output. Target: <500ms total for Tier 1, <700ms for Tier 2.

03

Three-Tier Model Routing

The nano classifier runs first on every request and decides which tier handles it. All calls go through LiteLLM proxy — never direct API calls.

04

Memory Stack

Four distinct stores, each optimised for a different query type. sqlite-vec lives inside kairos.db — no separate process.

05

Async Write-back Flow

Fires after every response via asyncio.create_task(). Never blocks the response path — user gets the reply first, memory catches up after.

06

Tool Execution & Safe Dispatch

Every tool call is schema-validated before execution. LLM output is never passed directly to subprocess. Tool result feeds back into LLM context for final answer synthesis.

07

Infrastructure & Deployment

Three Docker services orchestrated via Compose. Remote access via Tailscale — no port forwarding required. Three deployment phases: dev laptop → Hetzner VPS → self-hosted NAS.
